Use `Input` to build forms for your interface. This component provides a styled version of HTML `<input>`.

**Import component**

```lang=jsx
import { Input } from 'rimble-ui';
```

## Examples

Different ways to use the `Input` component in your product. _Note: these examples use the [Field](/components/rimble-ui/Field) component to render associated labels._

**Placeholder**

Adds placeholder text to suggest the type of input expected.

```js is=react-live
<Input type="text" required={true} placeholder="e.g. My hodl wallet" />
```

**Disabled input**

```js is=react-live
<Input type="text" placeholder="e.g. My hodl wallet" required={true} disabled />
```

**Optional input**

Renders "(Optional)" next to the label.

```js is=react-live
<Input type="text" placeholder="e.g. My hodl wallet" />
```

### Text input types

**Text input**

Brings up a standard keyboard.

```js is=react-live
<Input
  type="text"
  required={true}
  placeholder="e.g. 0xAc03BB73b6a9e108530AFf4Df5077c2B3D481e5A"
/>
```

**Email input**

Give you an @ on your keyboard.

```js is=react-live
<Input type="email" required={true} placeholder="you@example.com" />
```

**Password input**

Masks any input.

```js is=react-live
<Input type="password" required={true} />
```

**Search input**

Allows users to quickly remove a search term.

```js is=react-live
<Input type="search" required={true} placeholder="e.g. Ethereum" />
```

**URL input**

Brings up a url keyboard.

```js is=react-live
<Input type="url" required={true} placeholder="e.g. rimble.consensys.design" />
```

### Numerical input types

**Number input**

Opens the number page of your keyboard.

```js is=react-live
<Input type="number" required={true} placeholder="e.g. 123" />
```

**Range input**

See the [Slider](/components/rimble-ui/Slider) component to implement a range input type.

**Telephone input**

Brings up the telephone keyboard.

```js is=react-live
<Input type="tel" required={true} placeholder="e.g. +445575242519" />
```

### Time input types

**Date input**

Displays HTML5 date picker.

```js is=react-live
<Input type="date" />
```

**Time input**

Displays HTML5 time picker.

```js is=react-live
<Input type="time" />
```

**Month input**

Displays HTML5 month picker.

```js is=react-live
<Input type="month" />
```

**Week input**

Displays HTML5 week picker.

```js is=react-live
<Input type="week" />
```

### Other input types

**File input**

Displays HTML5 file picker.

```js is=react-live
<Input type="file" />
```

## Design guidelines

Some best practices for using `Input` in your product.

**Don't use placeholders instead of labels**

Placeholders disappear as the user enters the field. This makes reviewing form entries difficult as you lose the explanation of the expected input.

<DoDont>
  {{
    positive: (
      <Box>
        <Field label="ENS address">
          <Input
            type="text"
            width="100%"
            required={true}
            placeholder="e.g. aardvark.eth"
          />
        </Field>
      </Box>
    ),
    negative: (
      <Box>
        <Field>
          <Input
            type="text"
            width="100%"
            required={true}
            placeholder="ENS address"
          />
        </Field>
      </Box>
    )
  }}
</DoDont>
