Use the Button component when you want users to perform an action like connecting their Ethereum account or starting a transaction. It contains a label with an optional icon before or after the label. It maps to a `<button type='button'>` element and extends [Box](/components/rimble-ui/Box).

**Import component**

```lang=jsx
import { Button } from "rimble-ui";
```

## Examples

Different ways to use the `Button` component.

In special cases where you'd like to use a `<a>` styled like a `Button`, use `<Button as='a'></Button>` and provide an href.

**Default button**

Use to show users the primary action(s) they can take from the screen they're on.

```js is=react-live
<Button>Connect wallet</Button>
```

**Solid**

```js is=react-live
<Button>Connect wallet</Button>
```

**Outline**

```js is=react-live
<Button.Outline>Connect wallet</Button.Outline>
```

**Text**

```js is=react-live
<Button.Text>Connect wallet</Button.Text>
```

**Base**

Minimal styling. Used for composing buttons with more complex styles.

```js is=react-live
<Button.Base>Connect wallet</Button.Base>
```

**Different sizes**

```js is=react-live
<div>
  <Button size="small" mr={3}>
    Connect wallet
  </Button>

  <Button size="large">Connect wallet</Button>
</div>
```

**Disabled button**

Use to indicate to the user that they need to do something before they can perform the action.

```js is=react-live
<Button disabled>Connect wallet</Button>
```

**Icon**

Use to add visual reinforcement to an action. In most cases an icon should support button copy and not replace. Copy is less ambiguous than an icon.

```jsx is=react-live
<Flex alignItems={"center"}>
  <Button icon="Send" mr={3}>
    Send
  </Button>

  <Button icon="Send" iconpos="right" mr={3}>
    Send
  </Button>

  <Button icon="MoreHoriz" icononly />
</Flex>
```

**Loading button**

Use to give users reassurance that the action they want is happening. If possible, include copy that further explains what's happening like "Loading" or "Sending"

```js is=react-live
<Button>
  <Loader color="white" />
</Button>
```

**Link button**

Use when you want a primary action to navigate the user to a new page or area. Use action words as the button label.

_This also requires you to import the [Link](/components/rimble-ui/Link) component from the rimble-ui library._

```js is=react-live
<Button as="a" href="#!" target="\_blank" title="Learn more">
  Learn More
</Button>
```

**Link Button Outline**

```js is=react-live
<Button.Outline as="a" href="#!" target="\_blank" title="Learn more">
  Learn More
</Button.Outline>
```

**Success color style**

```js is=react-live
<Button variant="success">Success button</Button>
```

**Danger color style**

```js is=react-live
<Button variant="danger">Danger button</Button>
```

**Custom color**

```js is=react-live
<div>
  <Button mainColor="DarkCyan" marginRight={3}>
    Custom color button
  </Button>

  <Button.Outline mainColor="DarkCyan" marginRight={3}>
    Custom color button
  </Button.Outline>

  <Button.Text mainColor="DarkCyan">Custom color button</Button.Text>
</div>
```

## Design guidelines

Some best practices for using `Button` in your product.

**Don't just use icons**

<DoDont>
  {{
    positive: (
      <Box>
        <Text
          fontSize={1}
          mb={3}
        >
          Button copy helps with accessibility.
        </Text>
        <hr color="#f4f4f4"/>
        <Button icon="Send" mt={3}>
          Send Ether
        </Button>
      </Box>
    ),
    negative:
      <Box>
        <Text
          fontSize={1}
          mb={3}
        >
          Icons can be subjective. This relies on a user recognising this icon.
        </Text>
        <hr color="#f4f4f4"/>
        <Button icon="Send" mt={3} />
      </Box>
  }}
</DoDont>

**Don’t use `Button` for navigational links**

<DoDont>
  {{
    positive: (
      <Box>
        <Text
          fontSize={1}
          mb={3}
        >
          Use <Link href="/components/rimble-ui/Link" title="To Link component" target="_blank">Link</Link> or Button.Text for navigational links.
        </Text>
        <hr color="#f4f4f4"/>
        <Box mt={3}>
          <Link>Transaction history</Link>
        </Box>
      </Box>
      ),
    negative: (
      <Box>
        <Text
          fontSize={1}
          mb={3}
        >
          Buttons are for actions like starting a process, confirming choices or acknowleding something.
        </Text>
        <hr color="#f4f4f4"/>
        <Button mt={3}>Transaction history</Button>
      </Box>
      ),
  }}
</DoDont>

**Keep position consistent**

This will let users know where to look when they want to perform actions

**Consider visual hierarchy of buttons**

Don’t use too many buttons together on one piece of interface: prioritise the most important action. Use `Button.Text` and `Button.Outline` to give your actions visual hierarchy

## Content

Some best practices for writing `Button` microcopy.

[Content guidelines evidence](https://readabilityguidelines.myxwiki.org/xwiki/bin/view/User%20centred%20design/Links/)

**Explain what pressing the button will do**

<DoDont>
  {{
    positive: (
      <Box>
        <Text
          fontSize={1}
          mb={3}
        >
          Make sure the Button copy explains what pressing it will do.
        </Text>
        <hr color="#f4f4f4"/>
        <Button mt={3} size="medium">
          Send Ether
        </Button>
      </Box>
    ),
    negative: (
      <Box>
        <Text
          fontSize={1}
          mb={3}
        >
          Avoid generic terms that leave the next steps unclear.
        </Text>
        <hr color="#f4f4f4"/>
        <Button mt={3} size="medium">
          Submit
        </Button>
      </Box>
    )
  }}
</DoDont>

**Keep copy concise**

<DoDont>
  {{
    positive: (
      <Box>
        <Text
          fontSize={1}
          mb={3}
        >
          Remove articles that don't contribute to meaning (a, the, an).
        </Text>
        <hr color="#f4f4f4"/>
        <Button mt={3} size="medium">
          Add wallet
        </Button>
      </Box>
    ),
    negative: (
      <Box>
        <Text
          fontSize={1}
          mb={3}
        >
          Unnecessary articles will make your Button copy longer and harder to scan.
        </Text>
        <hr color="#f4f4f4"/>
        <Button mt={3} size="medium">
          Add a wallet
        </Button>
      </Box>
    )
  }}
</DoDont>

**Explain the next step, not the end goal**

<DoDont>
  {{
    positive: (
      <Box>
        <Text
          fontSize={1}
          mb={3}
        >
          Explain what the button will do at this stage in the flow. For example, if you have a button that launches a QR code for you to scan with your mobile...
        </Text>
        <hr color="#f4f4f4"/>
        <Button mt={3} size="medium">
          Show QR code
        </Button>
      </Box>
    ),
    negative: (
      <Box>
        <Text
          fontSize={1}
          mb={3}
        >
          Don't label your Button with the end result. For example, pressing the Button below won't scan the QR code, it will show it so you can then scan it.
        </Text>
        <hr color="#f4f4f4"/>
        <Button mt={3} size="medium">
          Scan QR code
        </Button>
      </Box>
    )
  }}
</DoDont>

**Lead with a verb and follow with a noun**

<DoDont>
  {{
    positive: (
      <Box>
        <Text
          fontSize={1}
          mb={3}
        >
          This will make it clear what will happen and to what when interacting with the Button.
        </Text>
        <hr color="#f4f4f4"/>
        <Box mt={3}>
          <Button mb={2} size="medium">Add wallet</Button>
        </Box>
        <Button size="medium">Send Ether</Button>
      </Box>
    ),
    negative: (
      <Box>
        <Text
          fontSize={1}
          mb={3}
        >
          Don’t rely on the rest of the interface to inform the user about what their action relates to. However, universally understood actions like "Save", "Next" and "Previous" may not need the noun.
        </Text>
        <hr color="#f4f4f4"/>
        <Box mt={3}>
          <Button mb={2} icon="AddCircle">Add</Button>
        </Box>
        <Button icon="Send">Send</Button>
    </Box>
    )
  }}
</DoDont>
