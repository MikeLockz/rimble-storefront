- tell the user that their transaction has started
- indicate progress

## Template anatomy

<Image
  width="100%"
  src="/images/transaction-started-anatomy.png"
  alt="An annotated image calling out the different elements of the template"
  mt={3}
  borderTop={1}
  borderWidth={3}
  borderColor="primary-light"
/>
<Heading as="h4"
  color="dark-gray"
  alignItem="flex-end"
  mt={4}
>
  1. Title
</Heading>
<Text
  color="mid-gray"
  mb={3}
>
  Reassures the user that their desired action is taking place. Use the subcopy to maybe explain the result of their action, for example "Your DAI is on its way to your wallet".
</Text>
<Heading as="h4"
  color="dark-gray"
  alignItem="flex-end"
>
  2. Progress bar
</Heading>
<Text
  color="mid-gray"
  mb={3}
>
  This should let the user know how complete their transaction is and give a sense of how much longer they'll need to wait. <RimbleGatsbyLink to="/guides/ux/transactions-set-user-expectations#how-we-did-it">Learn how we did this.</RimbleGatsbyLink>
</Text>
<Heading as="h4"
  color="dark-gray">
  3. Percentage and status message
</Heading>
<Text
  color="mid-gray"
  mb={3}
>
  Similar to above, this adds more detail to the transaction's progress. The status message simply provides reassurance at a glance that it's in progress.
</Text>
<Heading as="h4" color="dark-gray">
  4. Link
</Heading>
<Text
  color="mid-gray"
  mb={3}
>
  Gives you the opportunity to link interested users to more information, via Etherscan for example.
</Text>

<Heading as="h4" color="dark-gray">
  5. Close button
</Heading>
<Text
  color="mid-gray"
  mb={3}
>
  Allows the user to continue using your dApp. We recommend that if the user closes this, show the Transaction progress banner so the user can be kept updated while they go about their business. See <RimbleGatsbyLink to="/templates/transaction-progress-banner" title="To Transaction progress banner template">Transaction progress banner template</RimbleGatsbyLink>
</Text>
